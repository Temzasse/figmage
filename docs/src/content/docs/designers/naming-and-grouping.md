---
title: Naming and Grouping
description: Naming conventions that produce predictable token output for developers.
---

Use naming as your API contract between design and code.

## Naming rules

1. Use stable semantic names.
2. Avoid temporary or tool-generated labels.
3. Keep one naming pattern for each token family.

## Grouping rules

1. Use top-level groups that map to real usage contexts.
2. Keep parallel structures where relevant, such as `Light` and `Dark`.
3. Keep typography groups explicit, such as `Web` and `Native`.

## Suggested review cadence

1. Review naming before release.
2. Freeze naming during implementation windows.
3. Communicate planned renames before merging Figma changes.
